Controller

Application to manipulate the py3daxxel controller (setup, streaming).

usage: controller.py [-h] [-d DEVICE] {device,set,get,stream,decode} ...

command (required)

Run command with the loaded configuration.

command

Possible choices: device, set, get, stream, decode

Flags

General flags applied to all commands.

-d, --device

Specify the serial device to communicate with.

Default: “/dev/ttyACM0”

Sub-commands

device

Retrieve device information.

controller.py device [-h] [-l | -j | -r]

Named Arguments

-l, --list

List attached devices (human readable).

Default: False

-j, --json

List attached devices (machine readable as JSON).

Default: False

-r, --reboot

Performs a device reboot (reset).

Default: False

set

Configure output data rate, resolution and range.

controller.py set [-h]
                  [-o {ODR3200,ODR1600,ODR800,ODR400,ODR200,ODR100,ODR50,ODR25,ODR12_5,ODR6_25,ODR3_13,ODR1_56,ODR0_78,ODR0_39,ODR0_20,ODR0_10} | -s {FULL_RES_4MG_LSB,SCALED_10BIT} | -r {G2,G4,G8,G16}]

Named Arguments

-o, --outputdatarate

Possible choices: ODR3200, ODR1600, ODR800, ODR400, ODR200, ODR100, ODR50, ODR25, ODR12_5, ODR6_25, ODR3_13, ODR1_56, ODR0_78, ODR0_39, ODR0_20, ODR0_10

Set sampling rate.

-s, --scale

Possible choices: FULL_RES_4MG_LSB, SCALED_10BIT

Set sampling resolution.

-r, --range

Possible choices: G2, G4, G8, G16

Set sampling range.

get

Reads device parameters.

controller.py get [-h] [-v | -o | -s | -r | -u | -b | -a]

Named Arguments

-v, --version

Read firmware version from device.

Default: False

-o, --outputdatarate

Read sampling rate from sensor.

Default: False

-s, --scale

Read sampling resolution from sensor.

Default: False

-r, --range

Read sampling range from sensor.

Default: False

-u, --uptime

Read device uptime in ms.

Default: False

-b, --buffer

Read ring buffer statistic since last sampling-start.

Default: False

-a, --all

Read all parameter.

Default: False

stream

Starts or stops data streaming from device.

controller.py stream [-h] [-s [START] | -p]

Named Arguments

-s, --start

Starts streaming for n samples, If n=0 enables streaming until stop is requested (0 <= n <= UINT16_MAX).

-p, --stop

Stops current stream.

Default: False

decode

Connects to device and decodes input stream. The connection must be established before the data stream is started. While decoding, subsequent script calls with “stream” and “set” commands are allowed.

controller.py decode [-h] [--timeout TIMEOUT] [--wait] [- | --file [FILE]]

Named Arguments

--timeout

Duration in seconds the script waits until data is received (left unset or 0.0 waits forever). Raises exception otherwise.

Default: 0.0

--wait

Does not return on last decoded package but waits for the next stream. Times out if –timeout is not 0.0.

Default: False

-, --stdout

Prints streamed data to stdout. Script does not finish when stream stops and waits for subsequent runs.

Default: False

--file

Writes streamed data to file. Script finishes when stream is stopped. While decoding, simultaneous calls to output stream are allowed: start, stop and setup commands. Leave empty string for default fallback filename “stream-20240203-223833557.tsv”.

Recording

Records acceleration while sending (G-Code) to the printer.

usage: record_step.py [-h] --address ADDRESS [--port PORT] [--key KEY]
                      [--axis {x,y,z}] [--start START]
                      [--extragcode EXTRAGCODE] [--distance DISTANCE]
                      [--stepcount STEPCOUNT] [--gostart] [--returnstart]
                      [--autohome] [--device DEVICE]
                      [--outputdatarate {ODR3200,ODR1600,ODR800,ODR400,ODR200,ODR100,ODR50,ODR25,ODR12_5,ODR6_25,ODR3_13,ODR1_56,ODR0_78,ODR0_39,ODR0_20,ODR0_10}]
                      [--timelapse TIMELAPSE] [--timeout TIMEOUT] [--dryrun]
                      [- | --file [FILE]]

REST API

Octoprint arguments.

--address

OctoPrint address

--port

OctoPrint port (default 80)

Default: 80

--key

OctoPrint API key.

Default: 80

Trajectory

Trajectory generator

--axis

Possible choices: x, y, z

Axis to move.

Default: “x”

--start

Start point in mm to begin trajectory at.

Default: “200,140,20”

--extragcode

Extra G-Code to send before trajectory (i.e. input shaping: “M593 X F30 D0.15”).

Default: “”””

--distance

Distance in mm to travel back and forth.

Default: 20

--stepcount

Repeat travel pattern stepcount-times.

Default: 4

--gostart

Go to start position first, then start repetitions.

Default: False

--returnstart

Return to start point after last repetition.

Default: False

--autohome

Perform auto homing before trajectory.

Default: False

Controller

Acceleration microcontroller arguments.

--device

Controllers serial device node to communicate with.

Default: “/dev/ttyACM0”

--outputdatarate

Possible choices: ODR3200, ODR1600, ODR800, ODR400, ODR200, ODR100, ODR50, ODR25, ODR12_5, ODR6_25, ODR3_13, ODR1_56, ODR0_78, ODR0_39, ODR0_20, ODR0_10

Set specified sampling rate before sending G-Code.

Default: “ODR3200”

--timelapse

Timespan to record captured samples in seconds.

Default: 1.0

Output

Output arguments.

--timeout

Duration in seconds the script waits until data is received (0.0 waits forever). Raises exception otherwise.

Default: 0.0

--dryrun

Pretends to run but does not invoke either Octoprint nor controller.

Default: False

-, --stdout

Prints streamed data to stdout. Script does not finish when stream stops and waits for subsequent runs.

Default: False

--file

Specify output file (*.tsv). Leave empty for default filename.

Recording Sequences

Records a series of recording-steps samples while sending G-Code to the printer.

usage: record_step_series.py [-h] --address ADDRESS [--port PORT] [--key KEY]
                             [--axis {x,y,z,xy,xz,yz,xyz}] [--start START]
                             [--distance DISTANCE] [--stepcount STEPCOUNT]
                             [--sequencecount SEQUENCECOUNT]
                             [--fxstart FXSTART] [--fxstop FXSTOP]
                             [--fxstep FXSTEP] [--zetastart ZETASTART]
                             [--zetastop ZETASTOP] [--zetastep ZETASTEP]
                             [--device DEVICE]
                             [--outputdatarate {ODR3200,ODR1600,ODR800,ODR400,ODR200,ODR100,ODR50,ODR25,ODR12_5,ODR6_25,ODR3_13,ODR1_56,ODR0_78,ODR0_39,ODR0_20,ODR0_10}]
                             [--timelapse TIMELAPSE] [--timeout TIMEOUT]
                             [--dryrun] [--fileprefix FILEPREFIX]
                             [--directory DIRECTORY]

REST API

Octoprint arguments.

--address

OctoPrint address

--port

OctoPrint port (default 80)

Default: 80

--key

OctoPrint API key.

Default: 80

Trajectory

Trajectory settings

--axis

Possible choices: x, y, z, xy, xz, yz, xyz

Axis to move.

Default: “x”

--start

Start pont in mm to begin trajectory at.

Default: “200,140,20”

--distance

Distance in mm to travel back and forth.

Default: 20

--stepcount

Repeat travel back and forth (steps) stepcount-times.

Default: 2

Task

Capturing task repetitions with different parameters

--sequencecount

Repeats steps sequencecount-times.

Default: 1

--fxstart

Start frequency in Hz. See https://marlinfw.org/docs/gcode/M593.html

Default: 10

--fxstop

Start frequency in Hz.

Default: 80

--fxstep

Frequency increment in Hz.

Default: 10

--zetastart

Zeta damping factor (times 100). See https://marlinfw.org/docs/gcode/M593.html

Default: 0

--zetastop

Zeta damping factor (times 100).

Default: 25

--zetastep

Zeta damping factor increment (times 100).

Default: 5

Acceleration microcontroller arguments.

--device

Controllers serial device node to communicate with.

Default: “/dev/ttyACM0”

--outputdatarate

Possible choices: ODR3200, ODR1600, ODR800, ODR400, ODR200, ODR100, ODR50, ODR25, ODR12_5, ODR6_25, ODR3_13, ODR1_56, ODR0_78, ODR0_39, ODR0_20, ODR0_10

Duration in seconds the script waits until data is received (left unset or 0.0 waits forever). Raises exception otherwise.

Default: 0.0

Default: False

Specify prefix of output file (<prefix>-<run>-<timestamp>.tsv)

Default: “octo-capture”

Output path.

Default: ./test_data/

Data Visualization

usage: datavis.py [-h] [-f FILE] [-p] [-s] {algo} ...

command (required)

Run specified sub-command.

command

Possible choices: algo

Flags

General flags applied to all commands.

-f, --file

Input file name or regexp-pattern. Examples: “example.tsv”, “data/”,…)

Default: “example.tsv”

-p, --plot

Visualizes data

Default: False

-s, --save

Saves plots as PNG format.

Default: False

Sub-commands

algo

FFT Algorithm applied to input data.

datavis.py algo [-h] [-1 [{all,discrete,discrete_blackman}] | -2
                [{all,discrete}] | -3 [{all,trajectory}]]

Named Arguments

-1, --d1

Possible choices: all, discrete, discrete_blackman

Performs FFT 1D algorithms: discrete 1D, discrete 1D with Blackman window or both.

-2, --d2

Possible choices: all, discrete

FFT 2D algorithms: no algorithm implemented yet (TODO)

-3, --d3

Possible choices: all, trajectory

FFT 3D algorithms: compute trajectory from acceleration. Requires offset, gain and orientation calibration.
