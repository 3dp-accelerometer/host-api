A blocking decoder implementation.

This decoder allows to

  • tell the controller when sampling shall start, and

  • decode controllers’ stream (blocking).

The implementation is meant to be used threaded so that the decoding can be started (threaded) before sampling start is called.

Note: the serial device acquisition is performed at construction time.

Initialization

Acquires required resources for later interaction with controller.

Parameters:

  • controller_serial – i.e. “/dev/ttyACM0”

  • timelapse_s – how long to record

  • record_timeout_s – how long the controller shall record

  • sensor_output_data_rate – which sample rate the controller shall be configured

  • out_filename – decoded stream output file, leave None for not storage

  • do_dry_run – if true, will not invoke controller neither write output file but timing will as without dry-run

  • do_abort_flag – flag to externally shortcut the decoding loop
